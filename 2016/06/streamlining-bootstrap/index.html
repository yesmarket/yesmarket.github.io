Streamlining Bootstrap

2016-06-01

At my current workplace we need to smash out responsive pages at a pretty fast rate. As far as building the responsiveness, we’ve decided to specialise a bit whereby we have a guy working with the marketing team to build the responsive pages. These are subsequently given to the DEVs for data binding and wiring up to the back-end. The current approach was to build everything in Visual Studio, but this seemed like overkill. Don’t get me wrong; I love Visual Studio and have been using it since VS 2005, however it’s a pretty heavy IDE for the task at hand.

I played around with a few bootstrap editors like jetstrap and pingendo, but we require a pretty high level of customisation and to tell you the truth the quality of the generated markup from these tools was far from perfect. So back to ol’ faithful Sublime Text.

I’ve been playing around with gulp a lot lately - on a side note gulp > grunt, but I digress… I started playing around with gulp to build an awesome workflow for smashing out responsive HTML using bootstrap and SASS. Some of the Gulp plugins that I used include:

  • gulp-inject for injecting custom stylesheets and JavaScript into HTML pages.
  • wiredep for wiring up bower dependencies into HTML pages.
  • gulp-sass for automating the compilation of SASS into CSS.
  • gulp-autoprefixer for injecting vendor prefixes into CSS.
  • nodemon for monitoring changes in source code and automatically restarting express.
  • browser-sync for keeping multiple browsers & devices in sync when building websites.

All this was pretty cool stuff, especially when used in combination with Sublime plugins like Emmet, and bootstrap 3|4 snippets. But what is really cool was bringing the old mate yeoman along to the party.

I’d used yeoman generators before, so I had the idea of writing my own generator for scaffolding out my streamlined bootstrap workflow. Unfortunately there’s a few pre-requisites to all of this, so I’ll give you a quick rundown (in Windows PowerShell). Note: I installed all this stuff over a number of years, so hopefully I didn’t miss anything…

# install chocolatey
Set-ExecutionPolicy RemoteSigned
iwr https://chocolatey.org/install.ps1 | iex
# install a few chocolatey packages
cinst git -y
cinst poshgit -y
cinst nodejs -y
# install a few global npm packages
npm i -g bower
npm i -g gulp
npm i -g yo
# clone my yeoman generator
git clone https://github.com/yesmarket/generator-bootstrap
cd generator-bootstrap
npm link
# navigate to a location for builsing your streamlined bootstrap pages
cd [path-to-streamlined-bootstrap-dir]
# put the yeoman to work
yo bootstrap
# install all the bower dependencies
bower install
# open everything in sublime text
subl .
# create a new HTML page
yo bootstrap:newpage
# and finally run the gulp task runner to do all the things
gulp start
# profit :P
